“*Product discovery is essentially about separating good ideas from bad ideas”*

Usability testing involves using prototypes before we build the product, not at the end where any issues of significant waste.

You will need users to conduct user testing.

Find users on the internet, at tradeshows, etc

User testing should be conducted with high fidelity prototypes

Have an engineer present

Good PMs will know they will get the product wrong initially - nobody gets it right the first time”.

How to conduct the test:

- We want to learn whether the user has the problems we think they have, how they solves those problems today and what it would take for them to switch.
- Only the prototype can pass or fail, not the user
- Can users tell from your landing page what it is you do? Can they see what might be valuable or appealing?

*“Landing pages bridge the gap between expectations and what the product actually does”*

- Keep your users in use mode, rather than critique mode. What matters is that users are able to do the tasks they need to do.
- Keep quiet when you see someone struggling

Usability litmus test

1. The user got through the tasks with no problem and with no help
2. The user struggled and moaned a bit but eventually got through the tasks
3. The user got so frustrated that they gave up
- Use Parroting: “I see you’re looking at ____ on the right”
    - Tell them what they’re doing
