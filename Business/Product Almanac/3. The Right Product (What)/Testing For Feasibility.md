This is for your engineers - these questions need to be answered by them:

How do we know how to build this?

Do we have the skills on the team to build this?

Do we have enough time to build this?

Do we have architectural changes to build this?

Do we have on hand all the components we need to build this?

Do we understand the dependencies involved to build this?

Will the performance be acceptable?

Will it scale to the levels we need?

Do we have the infrastructure necessary to test and run this?

Can we afford the cost to provision this?

You need to give your engineers time to answer these questions

Question framing:

- Don’t ask if this can be done, but ask “what is the best way to do this and how long will it take?”

After this, an engineer may need to create a feasibility prototype to answer these questions. Many of our best product ideas are based on approached to solving problems that are now only possible due to new technology. Giving engineers time to answer these questions gives them autonomy to return with solutions that are far better than what you imagined.
