
## Loot analysis 


**Core Objective:** Standardizing the transformation of raw Sydney marketplace data into structured "Loot" using a Procedural Appraisal Engine.

---


The LAE evaluates every item on a scale of **1–100** across four vectors. These metrics drive the logic for rarity and user alerts.

### I. Value Score (Quantitative)
- **Definition:** Estimated liquid profit potential (Resale Margin).
- **Logic:** Comparisons against historical Sydney resale values for the specific brand/model.
- **Benchmarks:**
	- `0-20`: Zero resale (MDF/Particle board, broken items).
	- `21-60`: Mid-tier "Flippable" (IKEA, entry-level electronics).
	- `61-100`: High-ticket assets (Designer furniture, current-gen tech).

### II. Demand Score (Market Velocity)
- **Definition:** How fast the item will be claimed or sold.
- **Logic:** Volume of active 'Want' listings and category liquidity.
- **Benchmarks:**
	- `Low (1-30)`: Heavy/Niche (Pianos, massive wardrobes).
	- `High (71-100)`: High-liquidity (Office chairs, monitors, coffee machines).

### III. Condition Score (Qualitative Analysis)
- **Definition:** Physical health and visual integrity.
- **Logic:** Vision-analysis of images + Sentiment analysis of description (e.g., "scratched" vs "mint").
- **Benchmarks:**
	- `1-20`: For parts/Structural damage.
	- `81-100`: New in box / Pristine.

### IV. Overall Item Score (Weighted Power Level)
- **Formula:** `(Value * 0.4) + (Demand * 0.3) + (Condition * 0.3)`
- This weighted average dictates the **Rarity Tier**.

## 🌈 2. Rarity Tier Framework
Triggered by the **Overall Item Score**.

| Tier | Score Range | Logic | Color Code |
| :--- | :--- | :--- | :--- |
| **Common** | 1 – 25 | High-availability commodity items. | `#FFFFFF` |
| **Uncommon** | 26 – 45 | Reliable but generic/worn gear. | `#1EFF00` |
| **Rare** | 46 – 65 | Desirable brands in good condition. | `#0070DD` |
| **Epic** | 66 – 85 | High-end designer (e.g. King Living) or pro-tech. | `#A335EE` |
| **Legendary** | 86 – 100 | "The Grails." Extreme value or luxury status. | `#FF8000` |

## 📂 3. Data Schema (Output Fields)
The following fields are generated by the AI Agent and synced to the **Lootbook Database**.

### Metadata
- `item_id`: Unique source ID.
- `source_url`: FB Marketplace Link.
- `timestamp_appraised`: ISO-8601.

### Qualitative (AI-Generated)
- `loot_name`: Stylized product name (e.g., *Refined King Jasper*).
- `rarity_tier`: Calculated Tier Name.
- `red_text`: Witty, Borderlands-style flavor text.
- `ai_summary`: 2-sentence justification for the appraisal.

### Quantitative (Integers 1-100)
- `score_overall`: The "Power Level".
- `score_value`: Flipping potential.
- `score_demand`: Speed of claim.
- `score_condition`: Quality of state.

---

## 🛠️ 4. Workflow Integration Logic
1. **Scrape:** Apify retrieves raw data.
2. **Filter:** n8n checks Google Sheet for `source_url` to avoid redundant AI spend.
3. **Appraise:** AI Agent processes `Title` + `Description` + `ImageURL`.
4. **Log:** Structured JSON is appended to `Lootbook.xlsx`.
