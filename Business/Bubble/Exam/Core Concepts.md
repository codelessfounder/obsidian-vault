### **Database Structure and Relationships**

- Understand Bubble's **"Things"** (data types) and fields.
- Create/manage **data relationships** (one-to-many, many-to-many).
- Use **custom states** and **option sets** efficiently.

### **Workflows**

- Build/debug **action-based workflows** (e.g., navigation, data manipulation).
- Use **conditional workflows** effectively.
- Master **API workflows** for backend logic/automation.

### **Responsive Design**

- Use the **responsive editor** effectively.
- Handle various **device types** and screen sizes.
- Configure **container layouts** (row, column, alignment).

### **Data Privacy and Security Rules**

- Set up **privacy rules** to control data access.
- Prevent unauthorized data viewing/editing.
- Use **token-based authentication** and user roles.

### **Dynamic Data and Repeating Groups**

- Display/filter dynamic data in **repeating groups**.
- Use **search constraints** and sorting.
- Optimize large datasets with **pagination**.

### **API Connections**

- Set up/manage **API connectors** (RESTful APIs, JSON).
- Work with third-party services and test API calls.
- Handle API errors in workflows.

### **Conditionals**

- Create/debug **conditional logic** for UI/workflows.
- Use conditions for **element visibility** and interactivity.

### **Custom States**

- Use **custom states** for temporary data storage/UI behavior.
- Differentiate custom states from database fields.

### **Performance Optimization**

- Minimize heavy workflows; use efficient queries.
- **Cache data** via custom states/app-level storage.

### **Version Control and Collaboration**

- Use **versioning** for safe updates.
- Manage **team roles** and permissions effectively.
